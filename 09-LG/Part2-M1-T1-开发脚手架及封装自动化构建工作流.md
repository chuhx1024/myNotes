# 开发脚手架及封装自动化构建工作流

### 前端工程化
- 为什么
    - ES6 某些语法不支持
    - less scss ts 运行环境不能直接用
    - 模块化方式 提高可维护性 运行环境不能直接支持
    - 部署上线 压缩代码等 都是手动  麻烦
    - 多人协同时 无法硬性统一代码风格 
    - 从仓库 pull 回来的代码 质量无法保障
    - mock 数据的功能 工程化 也可以很好的解决

- 可以做什么
    - 创建项目 可以使用脚手架
    - 编码可以给项目内置 esList
    - 可以为项目开启 sourceMap
    - 提交代码 添加 git hook
    - 部署时 使用 CI/CD 实现自动发布

- 工程化 不等于 某个工具
    - 工程化 是对项目的整体规划和架构  而不是单单指某个工具 如 webpack

- 工程化的功劳要归功于 node.js
